A Node.js service receives incoming webhooks from a third-party provider. Each request carries an `X-Signature-256` header containing an HMAC-SHA256 digest of the raw request body. Valid events are acknowledged immediately and forwarded to an internal processing queue for async handling.

The handler must be robust to slow downstream processing and must not enqueue the same event more than once.

## Behaviors to cover

1. A request whose signature does not match its body is rejected with a 401 response; the body is not forwarded.
2. An event is written to the persistent queue before the HTTP 200 acknowledgment is returned to the sender.
3. The handler returns the HTTP response without waiting for downstream processing of the event to finish.
4. The response arrives within 3 seconds wall-clock time even when downstream processing of earlier events is slow and concurrent.
5. An event whose ID was already received is not written to the queue a second time.
6. When the queue reports it is full, the handler returns 503 with a Retry-After header.
7. The persistent queue actually enforces its configured capacity limit and rejects events that would exceed it.
