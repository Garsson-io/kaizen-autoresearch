A payment fraud detection service evaluates each transaction against two models. The scoring model applies configurable rules (transaction amount thresholds, velocity checks, geographic distance between consecutive transactions) and produces a numeric risk score. The risk model analyzes the transaction context — including merchant category, user purchase history, and device fingerprint — to predict whether the transaction is fraudulent. Transactions flagged by either model are held for manual review.

## Behaviors to cover

1. The scoring model assigns the correct risk score when a transaction exceeds the configured amount threshold for its merchant category.
2. The risk model identifies a fraudulent transaction pattern (e.g., rapid small purchases from different geographies) even when no single rule threshold is exceeded.
3. The velocity check in the scoring model correctly counts the number of transactions within the configured time window per user.
4. A transaction flagged by the risk model but not by the scoring model is still held for review — the models operate as an OR gate, not AND.
5. The combined output includes both the numeric risk score and the risk model's fraud probability, so the reviewer can see which model flagged the transaction and why.
