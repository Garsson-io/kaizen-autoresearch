A Node.js daemon watches a configured input directory, runs each new file through a transformation pipeline, and writes results to an output directory. It runs continuously and must handle unusual filesystem conditions without requiring a restart.

## Behaviors to cover

1. A file placed inside a subdirectory of the watched root is processed, not only files placed directly in the root directory.
2. The deduplication logic correctly discards a second processing request for the same file path within a configurable time window.
3. When the operating system emits multiple filesystem events for a single file write, the daemon processes the file exactly once.
4. If the watched directory is removed and then recreated while the daemon is running, the daemon resumes watching the new directory without a restart.
5. When multiple files arrive at the same time, they are processed in the order determined by their creation timestamps.
6. A file that is still being written to when the arrival event fires is not picked up for processing until writing is complete.
