A spreadsheet formula engine evaluates cell expressions, resolves cross-cell references, detects circular dependencies, and supports a library of built-in functions (SUM, VLOOKUP, IF, CONCATENATE, etc.). It persists workbooks to disk in a custom binary format and supports undo/redo for formula edits.

## Behaviors to cover

1. A cell containing `=SUM(A1:A10)` evaluates to the correct sum of the values in cells A1 through A10.
2. A cell referencing another cell that contains a formula (e.g., `=B1+1` where B1 is `=A1*2`) produces the correct cascaded result.
3. The engine detects a circular dependency (A1 references B1, B1 references A1) and returns a circular reference error instead of looping.
4. VLOOKUP with an exact match returns the correct value from the lookup table, and returns an error when the key is not found.
5. The IF function correctly handles nested conditions (e.g., `=IF(A1>10, IF(A1>20, "high", "medium"), "low")`).
6. CONCATENATE joins cell values with the correct separator and handles empty cells by skipping them rather than inserting blank strings.
7. A formula referencing a cell in a different sheet (e.g., `=Sheet2!A1+1`) resolves correctly when the referenced sheet exists.
8. When a referenced cell's value changes, all dependent formulas are recalculated in the correct topological order.
9. The undo operation after editing a formula restores both the formula text and all previously computed dependent values.
10. Saving a workbook to disk and loading it back produces identical cell values and formulas — no data loss or precision change in floating-point results.
