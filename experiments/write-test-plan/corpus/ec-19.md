An AI-powered code generation service accepts a natural language description of a function, generates the implementation using an LLM, runs the generated code against a provided test suite, and iteratively refines the implementation if tests fail up to 3 attempts before reporting failure.

## Behaviors to cover

1. The generated function signature matches the expected interface (parameter names, types, return type) specified in the description.
2. Orchestration executes attempts in order and returns immediately on the first passing attempt.
3. For a function that a single call usually fails, the live iterative loop can still produce a passing implementation within the allowed attempts.
4. When tests fail, the refinement prompt builder includes the failed test names and failure messages in the next LLM prompt.
5. The next live attempt changes implementation content in response to prior failure context rather than resubmitting the same code.
6. The service stops after 3 failed attempts and returns a structured error with all three attempts and their test results.
7. The full non-agentic pipeline wiring (generate call, test execution, attempt bookkeeping) works end to end with deterministic stubbed model outputs.
8. The full live loop (generate, test, refine, retest) solves at least one known hard function that fails in the first attempt.
