An AI-powered code generation service accepts a natural language description of a function, generates the implementation using an LLM, runs the generated code against a provided test suite, and iteratively refines the implementation if tests fail — up to 3 attempts before reporting failure.

## Behaviors to cover

1. The generated function signature matches the expected interface (parameter names, types, return type) specified in the description.
2. The generated implementation passes all provided test cases on the first or a subsequent attempt.
3. When tests fail, the refinement step includes the failure messages in the next LLM prompt so the model can learn from its mistakes.
4. The service stops after 3 failed attempts and returns a structured error with all three attempts and their test results.
5. The full loop — generate, test, refine, test again — produces a working implementation for a function that a single LLM call typically gets wrong.
