A data pipeline reads CSV files from an S3 bucket, validates each row against a schema, transforms valid rows into a normalized format, and writes the results to a PostgreSQL table. Invalid rows are written to a dead-letter queue for manual review.

## Behaviors to cover

1. Rows with missing required fields are rejected and appear in the dead-letter queue with the original row number.
2. The transformation correctly converts date strings in multiple formats (ISO 8601, US, European) to a single canonical format.
3. A CSV file with 500,000 rows completes processing within the configured timeout without running out of memory.
4. When the S3 bucket returns a transient 503 error, the pipeline retries with exponential backoff rather than failing the entire batch.
5. Rows successfully written to PostgreSQL are not duplicated if the pipeline is restarted mid-batch.
