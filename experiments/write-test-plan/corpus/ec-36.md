Three skills in a planning chain currently overwrite the same `plan` attachment slot. Deep-dive, evaluate, and implement each write artifacts with different purposes, so evaluate grounding is lost before implementation. A refactor introduces a separate `grounding` artifact while keeping `plan` for implementation output.

## Behaviors to cover

1. The evaluate-stage command stores grounding into a dedicated `grounding` artifact, not the `plan` artifact.
2. The implement-stage flow reads grounding when available and still writes its execution output to `plan`.
3. Deep-dive no longer overwrites `plan` as a redundant side effect when diagnosis already exists in issue content.
4. In a full multi-step agent run (`deep-dive -> write-plan/evaluate -> implement`), the final implementation plan reflects grounding produced earlier in the run.
5. When grounding is absent, implement falls back safely and still emits a valid plan artifact without corrupting other stored artifacts.
