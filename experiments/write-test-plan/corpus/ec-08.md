A TypeScript library wraps a third-party REST API that enforces per-account rate limits. Multiple callers within the same process share the same account quota. The library must handle quota exhaustion gracefully and expose a simple async \`call()\` method.

## Behaviors to cover

1. The wait time between retries doubles with each consecutive failure, up to a configured maximum.
2. After the API signals that the request rate is too high, the client replays the original request automatically after the appropriate wait.
3. When the account's total quota is exhausted, the client returns a typed \`QuotaExhaustedError\` to the caller rather than retrying indefinitely.
4. Requests issued concurrently from multiple async callers in the same process all draw from the same shared quota bucket.
5. When multiple requests are waiting to be sent, they are dispatched in the order they were originally submitted.
