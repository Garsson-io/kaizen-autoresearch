A Node.js migration runner discovers \`.sql\` files in a \`migrations/\` directory, applies them to a PostgreSQL database in version order, and records each applied migration in a \`_migrations\` metadata table. It must be safe to run multiple times and safe to run concurrently from separate processes.

## Behaviors to cover

1. Migrations are applied in ascending numeric order determined by the filename prefix.
2. A migration that has already been applied is not re-executed on a subsequent run.
3. When a migration statement fails, its changes are rolled back automatically and no further migrations in the same run are attempted.
4. When two instances of the runner start at the same time against the same database, each migration is applied exactly once across both instances.
5. On the very first run against a fresh database, the \`_migrations\` table is created before any migration file is applied.
