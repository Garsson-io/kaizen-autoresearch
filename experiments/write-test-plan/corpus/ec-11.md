A customer support chatbot accepts a user message and a conversation history, generates a response using an LLM, and logs the exchange to a database. The system has a tone-checking step that re-prompts the LLM if the initial response is flagged as too formal or too casual for the customer's tier.

## Behaviors to cover

1. The conversation history passed to the LLM is trimmed to the most recent 10 messages when the full history exceeds the context window.
2. The generated response addresses the customer's question rather than producing a generic or off-topic reply.
3. The tone re-check correctly identifies when a response is too formal for a casual-tier customer and triggers regeneration.
4. The full exchange (user message, generated response, any regenerations) is persisted to the database with correct timestamps.
5. When the tone check triggers regeneration, the second response replaces the first — the customer never sees the rejected draft.
