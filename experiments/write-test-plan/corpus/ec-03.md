A Python CLI tool reads a git repository and produces a summary of recent changes, broken down by category: files added, deleted, renamed, and modified. It outputs a structured report to stdout. The tool is designed to run in CI pipelines as well as in developer environments.

## Behaviors to cover

1. Renamed files appear in their own category and are not also counted as a deletion plus an addition.
2. Diffs that include binary files do not cause the tool to crash; binary files are counted in the output.
3. Running the tool on a repository that has no commits yet produces an empty report rather than an error.
4. The counts in the report match the actual change history of the repository being analyzed.
5. The output is identical whether the changes being analyzed are staged or unstaged.
