A log analysis tool ingests structured JSON logs from multiple microservices, correlates entries by trace ID, and produces a timeline view of each request's journey through the system. It detects anomalies such as unusually high latency or error spikes.

## Behaviors to cover

1. Log entries with the same trace ID are grouped together regardless of which microservice emitted them.
2. The timeline view orders entries by timestamp, correctly handling clock skew of up to 500ms between services.
3. A latency spike (p99 above 3x the baseline) is detected and flagged in the anomaly report.
4. Malformed JSON log entries are skipped without crashing the pipeline, and the count of skipped entries is reported.
5. The tool handles a burst of 10,000 log entries per second without dropping entries or falling behind.
