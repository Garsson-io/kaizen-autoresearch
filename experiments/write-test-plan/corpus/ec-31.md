A PostToolUse hook is responsible for opening the PR review loop whenever a command creates a pull request. An incident showed the hook missed `gh pr create` when the command included newline-separated variable assignments before the `gh` invocation. The hook classified the whole block as `not_a_trigger`, so no review gate state file was created.

## Behaviors to cover

1. Command segmentation treats newline-separated statements as separate segments, not one monolithic block.
2. A multi-line shell block with variable assignments followed by `gh pr create` is recognized as a PR-create trigger.
3. A similar multi-line block that ends with `gh issue create` is not incorrectly treated as a PR-create trigger.
4. When a PR-create trigger is detected, the hook can reconstruct and persist the PR identifier needed by the review loop state.
5. After PR-create detection, the review gate state is written in a form that the pre-tool enforcement hook can read and enforce.
