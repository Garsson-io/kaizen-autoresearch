A classification module accepts a document string and returns one label from a fixed taxonomy (e.g., `legal`, `technical`, `financial`, `other`). It delegates the classification decision to an external AI API and enforces a per-document token budget to control cost.

## Behaviors to cover

1. The request sent to the external API always includes the complete list of valid taxonomy labels.
2. When the API returns a label that is not in the taxonomy, the module returns `"unknown"` instead of forwarding the unexpected value to the caller.
3. For identical raw API responses, the module's normalization step always produces the same final label.
4. On repeated live API calls for the same document and taxonomy, the returned label remains consistent unless the provider response itself changes.
5. Request construction enforces the configured token budget before dispatch (for example truncate/reject according to policy).
6. Provider-reported token usage for a typical document stays within the configured per-call budget.
7. When the API signals that the request rate is too high, the module waits and retries rather than returning an error to the caller.
