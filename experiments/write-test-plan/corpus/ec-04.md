A classification module accepts a document string and returns one label from a fixed taxonomy (e.g., \`legal\`, \`technical\`, \`financial\`, \`other\`). It delegates the classification decision to an external AI API and enforces a per-document token budget to control cost.

## Behaviors to cover

1. The request sent to the external API always includes the complete list of valid taxonomy labels.
2. When the API returns a label that is not in the taxonomy, the module returns \`"unknown"\` instead of forwarding the unexpected value to the caller.
3. The classification result for the same document is the same across repeated calls.
4. The number of tokens consumed for a typical document stays within the configured per-call budget.
5. When the API signals that the request rate is too high, the module waits and retries rather than returning an error to the caller.
