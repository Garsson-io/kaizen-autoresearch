A Node.js extensibility framework loads plugin packages from a configured directory. Each plugin ships a \`manifest.json\` declaring its name, version, dependencies on other plugins, and lifecycle hook exports. The loader validates manifests, resolves load order, and calls each plugin's \`onLoad\` hook.

## Behaviors to cover

1. A plugin whose manifest is missing a required field is rejected before any code from that plugin is executed.
2. When plugin A declares a dependency on plugin B, plugin B's \`onLoad\` hook is called before plugin A's.
3. When two plugins each declare a dependency on the other, the loader reports the cycle and loads neither.
4. A plugin's \`onLoad\` hook is only invoked after the plugin's entry module has been imported and executed.
5. When a plugin throws an error during its \`onLoad\` hook, the loader continues initializing the remaining plugins that do not depend on it.
