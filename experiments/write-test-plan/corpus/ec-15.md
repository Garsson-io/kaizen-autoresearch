A REST API serves product search results. It accepts a natural language query, uses an embedding model to convert the query into a vector, performs a similarity search against a vector database, and returns the top-K results ranked by relevance. A re-ranking step uses an LLM to reorder the top results based on contextual relevance to the original query.

## Behaviors to cover

1. The embedding vector produced for a given query string has the correct dimensionality for the vector database index.
2. The similarity search returns results ordered by descending cosine similarity score.
3. The LLM re-ranking step changes the order of at least some results compared to the raw similarity ranking when the query has nuanced intent.
4. A query with no close matches in the database returns an empty result set rather than low-relevance noise.
5. The end-to-end latency from receiving the query to returning re-ranked results stays below 2 seconds for the median case.
