A scheduled Python script produces a weekly engineering status report. It collects data from three internal sources (a metrics endpoint, an issue tracker API, and a deployment log), generates a narrative summary from that data, and delivers the report by email to a configured recipient list.

## Behaviors to cover

1. The content passed to the summary-generation step includes data retrieved from all three sources.
2. When one data source is unavailable, the report is produced with a \`data unavailable\` note for that section rather than aborting.
3. The narrative summary accurately represents the key values present in the input data.
4. A complete report — from data collection through email delivery — reaches the configured recipient's inbox correctly.
5. When the email delivery service is unreachable, the generated report is saved to a local file and delivery is retried on the next scheduled run.
