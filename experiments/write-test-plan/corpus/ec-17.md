An autonomous email drafting assistant reads an incoming email thread, generates a context-aware reply draft using an LLM, checks the draft for factual consistency against a knowledge base, and presents the draft to the user for approval before sending.

## Behaviors to cover

1. The generated reply references specific points from the incoming email rather than producing a generic response.
2. The factual consistency check flags claims in the draft that contradict entries in the knowledge base.
3. When the knowledge base has no relevant entries, the draft includes a disclaimer rather than fabricating facts.
4. The approval workflow blocks sending until the user explicitly approves — no draft is sent automatically.
5. The full sequence — read thread, generate draft, check facts, present for approval — works end-to-end for a real email thread.
