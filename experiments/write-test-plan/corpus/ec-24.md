A real-time translation service accepts text in one language and returns a translation. It uses a neural machine translation API as the primary translator. When the translation API is unavailable or returns an error, the service falls back to a phrase-lookup dictionary stored in a local SQLite database. The service tracks translation quality by comparing character-level edit distance between the primary and fallback outputs when both are available.

## Behaviors to cover

1. The primary translator produces a grammatically coherent translation that preserves the meaning of the source text, not just a word-for-word substitution.
2. When the translation API returns a 503 error, the service transparently falls back to the dictionary-based translator without returning an error to the caller.
3. The dictionary-based fallback returns the correct pre-stored translation for phrases that exist in the SQLite database.
4. The quality tracking comparison correctly computes edit distance between primary and fallback outputs and stores the metric.
5. When both translators are available, the service returns the primary translation and logs the quality comparison in the background.
