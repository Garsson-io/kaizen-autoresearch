An e-commerce product recommendation engine generates personalized suggestions for users. It filters the product catalog based on availability and user preferences, applies a recommendation algorithm that weighs purchase history, browsing patterns, and similar-user behavior to rank products, paginates the results, and tracks which recommendations the user clicks for future training data.

## Behaviors to cover

1. Products marked as out-of-stock are excluded from the recommendation results regardless of their relevance score.
2. The user preference filter correctly excludes product categories the user has explicitly blocked in their settings.
3. The recommendation algorithm ranks products that match the user's recent browsing patterns higher than products with no browsing signal.
4. Pagination returns the correct page of results, with stable ordering — requesting page 2 twice returns identical results if the underlying data hasn't changed.
5. The recommendation algorithm surfaces at least one product from a category the user hasn't browsed before (serendipity requirement), avoiding a filter bubble.
6. A new user with no purchase or browsing history receives recommendations based on overall popularity rather than an empty result set.
7. Click tracking records the correct product ID, position in the list, and timestamp for each recommendation the user interacts with.
8. The response time for generating recommendations stays under 200ms for a catalog of 100,000 products.
9. When a recommended product is purchased by the user, the recommendation algorithm deprioritizes it in subsequent requests.
10. The recommendation results are deterministic for the same user state — requesting recommendations twice with no intervening user activity returns the same ranked list.
