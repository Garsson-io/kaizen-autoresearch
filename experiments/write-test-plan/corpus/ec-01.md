A Python CLI ships a YAML config file at \`~/.config/tool/config.yaml\`. Users can override any value by setting environment variables with the prefix \`TOOL_\`. The loader runs at startup; if loading fails the CLI exits with an actionable error message.

The tool also writes a defaults file on first run if no config directory exists yet.

## Behaviors to cover

1. When a required key is absent from both the file and the environment, the error message names the missing key by its exact config path (e.g., \`database.host\`).
2. When the environment variable \`XDG_CONFIG_HOME\` is set, the loader resolves the config file path relative to that directory rather than \`~/.config\`.
3. When an environment variable with the \`TOOL_\` prefix conflicts with a key in the config file, the environment variable value is used.
4. When the config file contains a YAML syntax error, the error message includes the line number where the problem was detected.
5. When the config directory does not exist at startup, the loader creates it and writes a defaults file before proceeding.
