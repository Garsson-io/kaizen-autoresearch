A CI/CD pipeline lint step reads all changed files in a pull request, runs static analysis rules, and posts a review comment on the PR with findings grouped by severity. The step runs as a GitHub Action and must complete within the Actions timeout.

## Behaviors to cover

1. Only files changed in the pull request are analyzed — unchanged files are not scanned.
2. Findings are grouped by severity (error, warning, info) in the posted PR comment.
3. A pull request that touches 200 files completes analysis within the 10-minute Actions timeout.
4. When the linter process crashes on a malformed file, the remaining files are still analyzed and results posted.
5. Duplicate findings on the same line from overlapping rules are deduplicated in the output.
