A team added behavioral smoke tests for skill changes, but the tests included phase instructions copied from SKILL.md directly in the prompt. Review flagged this as tautological: the test proved the model follows explicit instructions, not that the skill context causes the behavior. The team wants reliable tests that fail when the skill content regresses.

## Behaviors to cover

1. A static prompt-template lint check rejects phase-specific instruction leakage copied from SKILL.md.
2. The fully assembled prompt produced by the real test harness (after interpolation and fixtures) still contains no phase-instruction leakage.
3. The test invocation loads skill context from plugin/skill files and validates behavior from that context rather than from hard-coded prompt rules.
4. The harness can run the same user-style prompt against an intentionally degraded or missing SKILL.md variant without changing test prompt text.
5. When the relevant SKILL.md section is removed or degraded, the behavioral test fails under the same user-style prompt.
6. Raw model output is captured to a diagnostic artifact before any JSON parsing or extraction logic runs.
7. Assertions enforce structural behavior constraints (for example schema paths and role-specific field placement), not only keyword presence checks.
8. Under live model execution, structural assertions catch regressions that a keyword-only check would miss.
