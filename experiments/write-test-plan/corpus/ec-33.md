In a PR review cycle, a summary comment declared all dimensions PASS even though stored per-dimension findings still contained FAIL verdicts. The failure was traced to manual summary composition without rerunning review agents after fixes. The gate trusted the summary text and did not cross-check stored findings.

## Behaviors to cover

1. Storing a PASS review summary is blocked when any stored dimension finding for the same round has a FAIL verdict.
2. Stop-gate verdict computation reads per-dimension stored findings directly and does not rely only on hand-written summary text.
3. Manual summary text overrides require an explicit override flag and produce an auditable warning record.
4. After Phase 4 fixes, the review loop cannot finalize PASS unless a fresh review-agent round is executed on the updated diff.
5. If review rounds are exhausted with unresolved MUST-FIX findings, the flow emits an escalation outcome instead of PASS.
