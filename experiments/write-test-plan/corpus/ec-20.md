A vulnerability scanner reads a project's dependency manifest (package.json, requirements.txt, etc.), queries a vulnerability database API for known CVEs, and generates a report sorted by severity. When critical vulnerabilities are found, it automatically creates a pull request that bumps the affected dependency to the patched version.

## Behaviors to cover

1. All direct and transitive dependencies are extracted from the manifest and included in the vulnerability check.
2. The vulnerability database query returns results that match the exact version ranges in the manifest, not just the package name.
3. The severity sorting places Critical above High above Medium above Low, with CVEs of the same severity sorted by CVSS score.
4. The auto-generated pull request bumps only the affected dependency and does not modify unrelated entries in the manifest.
5. When the patched version introduces a breaking change (major version bump), the scanner flags it for manual review instead of auto-creating the PR.
