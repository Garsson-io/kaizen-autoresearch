A Python automation reads a pull request, identifies potential problems in the changed code (bugs, missing error handling, style issues), posts inline review comments at the relevant positions, and creates a top-level PR comment that summarizes all findings.

## Behaviors to cover

1. The review input is scoped to the lines that changed, not the full content of each modified file.
2. Inline comments are attached to the correct positions in the diff view on the pull request.
3. The agent identifies at least one real defect in a pull request that contains a known introduced bug.
4. Every finding listed in the top-level summary corresponds to a posted inline comment ID from the same run.
5. No inline finding is dropped from the final summary payload before posting.
6. The automation sequence (read diff, post inline comments, post summary) succeeds for a real pull request when provided a fixed precomputed finding set.
7. The full live run with model-generated findings (read diff, analyze, post inline comments, post summary) completes correctly for a real pull request without manual intervention.
