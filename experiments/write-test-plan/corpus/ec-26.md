A job scheduling engine accepts cron expressions and manages recurring task execution. It parses cron strings into schedule objects, computes the next N run times from a given start time, detects overlapping schedules that would cause resource contention, and handles timezone-aware scheduling including DST transitions.

## Behaviors to cover

1. The cron parser rejects invalid expressions (e.g., month 13, day 32, malformed syntax) with a descriptive error message.
2. The next-run-time calculator returns correct timestamps for standard cron expressions, including edge cases like "last day of month" and "nearest weekday."
3. The overlap detector identifies when two schedules would fire within the configured minimum gap (e.g., 5 minutes) and reports the conflicting time windows.
4. Timezone-aware scheduling produces correct run times across a DST transition — a job scheduled for 2:30 AM handles the spring-forward gap and fall-back ambiguity correctly.
5. When a scheduled job's computed next run time has already passed (e.g., system was down), the engine determines whether to skip or backfill based on the job's configured missed-run policy.
